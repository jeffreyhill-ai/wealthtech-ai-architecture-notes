# WealthTech AI Architecture Notes

Short architecture notes on trusted financial AI, provider data, and advisor workflows.

The central idea is simple: the next generation of wealthtech will not be defined by AI features pasted onto existing dashboards. It will be defined by systems that can turn fragmented evidence into governed, source-linked, reviewable financial intelligence.

## Notes

- [Why financial AI needs provider-aware data contracts](notes/provider-aware-data-contracts.md)
- [Why advisor AI needs review queues](notes/advisor-review-queues.md)
- [Why dashboards are downstream of evidence workflows](notes/dashboards-are-downstream.md)

## Themes

- Financial data provider realities should shape AI product architecture.
- Source grounding, lineage, and entitlement awareness belong in the product model.
- Advisor-facing AI should make uncertainty visible as work, not hide it inside polished language.
- Dashboards are useful after the evidence has been structured, reviewed, and approved.
